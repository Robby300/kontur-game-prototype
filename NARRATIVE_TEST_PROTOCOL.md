# NARRATIVE_TEST_PROTOCOL — закрытый протокол NG1

Статус: `v0.4.0-n1-draft.1`. Документ предназначен ведущему и владельцу;
участнику не показываются этот файл, граф, пакеты, presentations, скрытые факты
и назначенный риск до прохождения.

## 1. Границы теста

Проверяется воспроизводимость заранее написанной narrative-подачи, а не
production-код, LLM, Telegram, 3D или механика v0.3. Ведущий не является
мастером-автором: он находит строку в `NARRATIVE_SCENARIO_V04.md` и показывает
соответствующий текст из `NARRATIVE_PRESENTATIONS.md` дословно.

Участник получает только:

> «Это короткий неканонический сценарий о группе КОНТУРА. Смотрите карту перед
> каждой сценой, думайте вслух, выбирайте один из предложенных вариантов. Здесь
> нет правильного морального ответа. Если вам неясно, что происходит, скажите
> это вслух; ведущий не будет объяснять или советовать».

До начала получить информированное согласие: цель исследовательского теста,
ожидаемые 15–30 минут, право остановиться без объяснения, отсутствие записи
голоса и экрана без отдельного явного разрешения. Отказ от записи не влияет на
участие.

## 2. Версии и заморозка

До технического пилота используются `scenarioVersion = v0.4.0-n1` и
`contentVersion = v0.4.0-n1-draft.1`.

1. Провести один технический пилот; он не входит в результат NG1.
2. Зафиксировать только наблюдения о ясности текста, карте и следовании
   протоколу.
3. Разрешена максимум одна правка формулировок. Нельзя менять граф, ресурсы,
   `choiceId`, набор пакетов, риск, терминалы или число путей.
4. После возможной одной правки присвоить новый `contentVersion`, запретить
   дальнейшие изменения и вычислить внешний SHA-256 пакета из четырёх файлов:
   `NARRATIVE_SCENARIO_V04.md`, `NARRATIVE_SCENE_PACKETS.md`,
   `NARRATIVE_PRESENTATIONS.md`, `NARRATIVE_TEST_PROTOCOL.md`.
5. Внешне записать версию, дату, имена файлов и SHA-256 в журнал владельца или
   утверждённом Git commit. Не добавлять контрольную сумму внутрь хешируемых
   файлов.

Пример внешней команды после заморозки:

```bash
sha256sum NARRATIVE_SCENARIO_V04.md NARRATIVE_SCENE_PACKETS.md \
  NARRATIVE_PRESENTATIONS.md NARRATIVE_TEST_PROTOCOL.md
```

## 3. Назначение риска

До каждой пригодной сессии владелец записывает пару в закрытом журнале. Ведущий
не заменяет её после выбора маршрута.

| Сессия | Назначенный случай | Полоса |
| --- | --- | --- |
| Пригодная 1 | `2+3` | `<= 6` |
| Пригодная 2 | `3+4` | `7–9` |
| Пригодная 3 | `5+5` | `>= 10` |

Если сессия прервана, не завершена или ведущий вмешался, она помечается
непригодной. Её случай не переносится молча: владелец отдельно назначает пару
для замещающей сессии и записывает причину замены.

## 4. Пошаговая инструкция ведущему

1. Открыть только `NARRATIVE_PRESENTATIONS.md`, `NARRATIVE_SCENARIO_V04.md` и
   закрытый журнал назначений. Не показывать файловую систему участнику.
2. Показать участнику вводный текст из раздела 1 и `NRV04-PR01` целиком.
3. Записать выбранный `choiceId` буквально; не интерпретировать фразу участника
   как другой вариант. При необходимости повторить только названия доступных
   вариантов из presentation.
4. По переходной таблице показать `PR02` или `PR03`.
5. После выбора маршрута сверить назначенный случай и показать ровно один из
   `PR04`…`PR09`. Сначала дочитать событие, затем числа кубиков и короткую
   причинную сводку, как написано в presentation.
6. После следующего `choiceId` показать `PR10`…`PR13` по таблице терминалов.
7. Только после того, как участник ответит на исследовательские вопросы ниже,
   можно показать подробную строку его пути из `NARRATIVE_SCENARIO_V04.md`.
8. Поблагодарить участника и заполнить форму наблюдений. Не объяснять скрытые
   факты, не оправдывать автора и не спорить с оценкой.

Ведущему запрещено объяснять географию, понятия, цену, правила проверки или
последствия; перефразировать presentation; подсвечивать «разумный» выбор;
добавлять реплику, маршрут, объект, персонажа, ресурс либо исход; менять
назначенный риск. Разрешено лишь прочитать заново уже показанный фрагмент.

## 5. Форма фиксации одной сессии

```text
Участник / псевдоним:
Дата:
Версии scenario/content:
Внешний SHA-256 пакета:
Согласие получено: да / нет
Отдельное разрешение записи: нет / голос / экран / оба
Назначенный риск: 2+3 / 3+4 / 5+5
Фактический путь packetId + choiceId:
Время начала / окончания / длительность:
Завершил без помощи: да / нет
Было вмешательство ведущего: нет / да, точная цитата и момент
Был отказ или незавершение: нет / да, причина со слов участника

До подробного отчёта спросить дословно:
1. «Где были КОНТУР, группа, Аша и кассета в ключевой сцене?»
2. «Что такое кассета и почему она была важна?»
3. «Какое решение было самым трудным? Почему?»
4. «Какой риск был у маршрута и что, по-вашему, означал результат кубиков?»
5. «Хотелось бы увидеть следующий день именно этого мира? Почему?»
6. «Это ощущалось как история или как меню ресурсов? Что это изменило?»

Ответы без подсказок:
Понимание географии: верно / частично / неверно
Понимание кассеты: верно / частично / неверно
Назван конкретный трудный выбор: да / нет
Риск и результат объяснены: понятно / случайно / скрытая таблица / иное
Желание продолжить: да / нет / неопределённо
История, а не меню: да / нет / смешанно
Наблюдения ведущего без интерпретации:
```

## 6. Решение NG1

Три пригодные сессии должны длиться 15–30 минут. `NG1` открываются только если
одновременно выполнены все критерии из `NARRATIVE_PLAYTEST_REPORT.md`:

- все трое завершили сценарий за 15–30 минут без объяснения автора;
- минимум двое верно объяснили географию ключевой сцены;
- минимум двое своими словами объяснили кассету;
- минимум двое назвали конкретный трудный выбор;
- минимум двое захотели увидеть следующий день именно своего мира;
- никто не описал исход как случайную смену текста или скрытую таблицу.

Форма решения:

```text
Технический пилот: проведён / не проведён
Число использованных правок после пилота: 0 / 1
Пакет заморожен внешним SHA-256: да / нет
Пригодные сессии: 0 / 1 / 2 / 3
Критерии NG1: [для каждого да / нет, с номером сессии]
Решение: NG1 PASS / NG1 FAIL / данных недостаточно
Подпись владельца и дата:
```

`NG1 PASS` разрешает только отдельный выбор владельца между authored и hybrid
renderer. Он не открывает P2, P3, Telegram, сервер, LLM в runtime, новый
production-день или полноценную 3D-разработку.

## 7. Предстартовый аудит ведущего

- [ ] У каждого из 13 пакетов есть готовый presentation и `SceneLayout` до
  текста.
- [ ] Каждая строка `packetId + choiceId + riskCase` однозначна.
- [ ] Все 24 пути заканчиваются на `T01`…`T04`; на каждом есть одна проверка.
- [ ] Пары `2+3`, `3+4`, `5+5` дают разные публичные события и следующий выбор.
- [ ] Ни один текст до броска не раскрывает коррозию, кабельный направляющий
  или сухой обход.
- [ ] Цена каждого выбора совпадает с описанным терминальным состоянием.
- [ ] В инструкциях нет фразы, разрешающей ведущему дописывать по ситуации.
- [ ] Выполнены Markdown-проверки и `git diff --check`.
